# **Getting Started**

Welcome to the **Docs Framework** built with Next.js and MDX. This framework is designed to give you **full control** over your documentation without relying on heavy abstractions.


**🚀 What You Get**

* File-based documentation system  
* Automatic sidebar generation  
* \_meta.mdx for folder control  
* Built-in components (Callout, Tabs, etc.)  
* Light / Dark theme support  
* Fully customizable UI


**📂 Project Structure**

`content/docs/`  
  `getting-started.mdx`  
  `guides/`  
    `_meta.mdx`  
    `intro.mdx`


**⚙️ Configuration**

All behavior is controlled via /lib/config.ts:  
`export const docsConfig = {`  
  `home: ["getting-started"],`  
  `docs: {`  
    `contentDir: "content/docs",`  
    `basePath: "/docs",`  
  `},`  
`}`


**✍️ Writing Docs**

Create .mdx files inside content/docs/.  
**Example (intro.mdx):**  
`---`  
`title: Introduction`  
`order: 1`  
`---`

`# Introduction`  
`Welcome to your docs!`


**🧩 Components**

### **Callout**

Informational and warning messages to highlight important details.

### **Tabs**

Switch between different code snippets or instructions (e.g., cURL vs Node.js).


**🔗 Links**

Use internal links like \[Go to API\](./api/overview) or external links like \[Visit GitHub\](https://github.com).


**🧭 Navigation**

* Sidebar is generated automatically  
* Breadcrumbs are handled for you  
* Previous / Next navigation is included


**🎨 Theming**

Includes Light mode, Dark mode, and System preference detection.


**📁 \_meta.mdx**

Control folder behavior and ordering:  
`---`  
`title: Guides`  
`order: 2`  
`items:`  
  `- intro`  
  `- setup`  
`---`


**🛠️ Customization**

Override components in /lib/config.ts:  
`components: {`  
  `Navbar: CustomNavbar,`  
  `Footer: CustomFooter,`  
`}`
