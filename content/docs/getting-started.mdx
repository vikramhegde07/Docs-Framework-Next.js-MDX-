# Getting Started

Welcome to **Docs Framework** — a lightweight, flexible documentation system built with Next.js and MDX.

This guide will help you get your documentation site up and running quickly.

---

## 🚀 What is Docs Framework?

Docs Framework is designed to:

* Render **MDX-based documentation**
* Support **local & remote content sources**
* Provide a **clean docs UI (sidebar, breadcrumbs, navigation)**
* Be **fully static-export compatible**

---

## 📦 Project Structure

By default, your docs live inside:

```txt
content/docs/
├── _meta.mdx
├── getting-started.mdx
```

You can organize content however you want:

```txt
content/docs/
├── guides/
│   ├── _meta.mdx
│   ├── intro.mdx
│   └── advanced/
│       └── auth.mdx
```

---

## ✍️ Writing Your First Doc

Create a new file:

```txt
content/docs/intro.mdx
```

Add content:

```mdx
---
title: Introduction
description: Learn what this project is about
---

# Introduction

Welcome to your documentation 🚀
```

---

## 🧭 Navigation & Sidebar

Docs Framework automatically:

* Builds sidebar from folder structure
* Supports ordering via `_meta.mdx`
* Generates breadcrumbs
* Adds previous/next navigation

---

## 🧠 Metadata (SEO)

Each file supports frontmatter:

```mdx
---
title: My Page
description: Page description
keywords: [docs, mdx]
ogImage: /og.png
---
```

---

## 🗂 Folder Configuration (`_meta.mdx`)

You can control sidebar behavior:

```mdx
---
title: Guides
order: 2
items:
  intro:
    title: Introduction
  advanced:
    title: Advanced Guide
---
```

---

## 🌐 Content Sources

You can switch between:

### Local (default)

* Reads from `content/docs`

### GitHub (remote)

* Fetches docs from a repository
* Enables content editing without redeploying code

Configured in:

```ts
docs: {
  source: {
    type: "local"
  }
}
```

---

## ⚡ Static Export

Docs Framework supports full static export:

```bash
npm run build
```

Your site will be generated as static HTML.

---

## 🧩 Custom Components

You can use built-in MDX components:

```mdx
<Callout type="info">
This is an info message
</Callout>

<Tabs>
  <Tab title="JavaScript">
    console.log("Hello")
  </Tab>
</Tabs>
```
