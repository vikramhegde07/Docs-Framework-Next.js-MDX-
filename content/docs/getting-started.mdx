# Getting Started

Welcome to the **Docs Framework** built with Next.js and MDX.
This framework is designed to give you **full control** over your documentation without relying on heavy abstractions.

---

## 🚀 What You Get

* File-based documentation system
* Automatic sidebar generation
* `_meta.mdx` for folder control
* Built-in components (Callout, Tabs, etc.)
* Light / Dark theme support
* Fully customizable UI

---

## 📂 Project Structure

```txt filename="content structure"
content/docs/
  getting-started.mdx
  guides/
    _meta.mdx
    intro.mdx
```

---

## ⚙️ Configuration

All behavior is controlled via `/lib/config.ts`:

```ts filename="/lib/config.ts"
export const docsConfig = {
  home: ["getting-started"],
  docs: {
    contentDir: "content/docs",
    basePath: "/docs",
  },
}
```

---

## ✍️ Writing Docs

Create `.mdx` files inside `content/docs/`.

### Example (`intro.mdx`)

```mdx filename="intro.mdx"
---
title: Introduction
order: 1
---

# Introduction

Welcome to your docs!
```

## 📁 `_meta.mdx`

Control folder structure, ordering, and file titles.

---

### Example

```mdx filename="_meta.mdx"
---
title: Guides
order: 2
items:
  intro:
    title: Introduction
  setup:
    title: Setup Guide
---
```

---

### What it does

* Sets folder title → `Guides`
* Controls order → `intro`, `setup`
* Overrides file titles → cleaner sidebar labels

---

### Simple Ordering

```mdx
---
items:
  - intro
  - setup
---
```

---

### File Title Override

```mdx
---
items:
  getting-started:
    title: Getting Started
---
```

---

<Callout type="info">
You can use `_meta.mdx` at any level, including the root `content/docs` directory.
</Callout>
